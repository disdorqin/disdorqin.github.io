# Site Plan — disdorqin.github.io

This document outlines the intended structure and content areas for the personal research portal.

## Current State

The site currently serves a single-page HTML profile with basic information about the Cyber Energy Research Lab. It links to key GitHub projects but has minimal navigation and no content sections.

## Proposed Sections

### 1. Research Profile

- **Research interests**: Time series forecasting, power markets, energy systems, automated research workflows
- **Academic affiliation**: CS Student (indicate university and program)
- **Brief bio**: 2-3 sentences summarizing research direction
- **Publication list**: For future papers or preprints (arXiv links when available)

### 2. Project Showcase

Highlight selected open source projects with:

- **DARIS**: Automated research workflow system
  - Description, architecture overview, live demo / screenshot
  - Link: `https://github.com/disdorqin/DARIS`
- **research-claw**: Multi-agent research paper generator
  - Description, pipeline diagram, example output
  - Link: `https://github.com/disdorqin/research-claw`
- **TSPLab**: Time series prediction experiment lab
  - Description, supported models, benchmark results
  - Link: `https://github.com/disdorqin/tsplab`
- **CampusShelf**: Campus learning resource platform
  - Description, tech stack, deployment guide
  - Link: `https://github.com/disdorqin/CampusShelf`
- **WeMemoir**: WeChat memoir generation tool
  - Description, privacy-first design, input formats
  - Link: `https://github.com/disdorqin/wememoir`

### 3. Technical Blog (Optional)

Short write-ups about:

- Time series forecasting techniques and experiments
- Building automated research pipelines
- Lessons from open source project maintenance
- Power market data analysis notes

### 4. GitHub Integration

- Dynamic contribution graph
- Pinned repository widgets with stats
- Language distribution breakdown
- Trophy / achievement display (if appropriate)

### 5. Contact & Links

- GitHub profile
- LinkedIn (if applicable)
- Email (with spam protection)
- Academic profiles (Google Scholar, ORCID — when available)

## Implementation Notes

- **Static site**: No server-side rendering needed; pure HTML/CSS/JS
- **Dark theme**: Consistent with the current profile README's neon/cyber aesthetic
- **Responsive**: Works on desktop and mobile
- **Fast loading**: Minimal dependencies, no heavy frameworks
- **Accessibility**: Semantic HTML, alt text on images, proper heading hierarchy

## Content Principles

- Only include **real, verifiable** projects and achievements
- Do not claim publications or results that don't exist yet
- Keep the about text up to date with current academic status
- Link to actual GitHub repos rather than duplicating content

## Roadmap

- [ ] Phase 1: Research profile + project showcase (current focus)
- [ ] Phase 2: Technical blog with 3-5 initial posts
- [ ] Phase 3: Live demo pages for key projects
- [ ] Phase 4: CV / resume page
