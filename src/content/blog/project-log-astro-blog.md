决定自己搭博客之前，我用过不少平台。从最早的新浪博客、到后来的 Medium、知乎专栏、甚至自己用 Hexo 捣鼓过一阵子。绕了一大圈，发现每一样都不太对劲——不是定制空间太少，就是写起来不像自己的地方。

这次我想要的东西很纯粹：纯静态，尽可能轻，不需要读者加载一堆 JavaScript 才能读一篇文章。还有一点，我希望写文章和写代码是两件事——我不想每次发篇博客都要打开编辑器、提交 PR、等构建。我就想打开一个网页，填内容，保存，完事。

Astro 恰好站在这些需求的交汇点上。它的群岛架构让文章页面默认零 JS，首屏加载只送出 HTML 和 CSS，快得离谱。内容集合功能用 Zod schema 约束每篇文章的 frontmatter——缺字段、类型写错，构建期就报错。这意味着我不需要等到上线才发现"哦，忘了写 description"。

选择 Astro 的另一个理由是它和我的直觉方向一致：越是内容型的站点，越应该把运行时负担从读者身上移走。你写一篇文章，读者拿到的是纯内容；你说一句话，对方听到的是那句话本身——而不是先加载一套对话框架再去理解你的意思。Astro 做的，本质上就是这件事。

至于写作流程，我把它交给了 Pages CMS。它读仓库里的配置，在浏览器里给我一个表单界面：填标题、选分类、上传封面，保存即提交到 GitHub。再由 GitHub Actions 自动构建部署。从"写完"到"上线"，不需要碰命令行。

这套流程跑通的那天，我在留言板写了一条测试评论，盯着浏览器里刚刚部署上去的新文章，觉得这一切终于对了。它不是一个功能完备到无所不包的博客系统——它只是恰好给我留出了足够的空间，让我愿意在里面写字。

下一个想补的是归档时间线，把文章按年月梳理出来。慢慢来。
